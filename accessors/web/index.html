Accessors

What are Accessors?

An accessor is a software component that wraps a (possibly remote) device or service in an actor interface. An actor actor is a software component that reacts to streaming input events and produces streaming output events.
Accessors are defined in a JavaScript file that includes a specification of the interface (inputs, outputs, and parameters) and an implementation of the functionality (reactions to inputs and/or production of outputs). Any JavaScript file that conforms with the accessor specification defines an accessor class.

An instance of an accessor is created by a swarmlet host that evaluates the JavaScript in the accessor definition. At this time, there are at least three accessor hosts compatible with accessor specification 1.0:

The host uses the accessor as if it were a local source and/or sink for data and/or commands. An accessor host is to the Internet of Things what a browser is to the Internet. It renders a remote service by locally executing a proxy for that service.

The wiki page for this workgroup is the place to go to participate in the design of the accessor mechanism.

The TerraSwarm project has developed a "version 0.1" accessor design and a suite of accessor hosts that should be viewed as concept demonstrations. We are further developing the design. Ptolemy II (the Development Version in the SVN repository) is currently the most complete prototype of an accessor host. In Ptolemy II, the accessors listed here can be instantiated using the File-Import-Import Accessor menu item, and can then be incorporated into a Ptolemy II model. For a tutorial on Ptolemy II, see the Ptolemy Book (a free download). To view the accessor script from within Ptolemy II, double click on the accessor instance, select Preferences, and enable "expert mode."

Demos

To run the following models, save them to some directory, and open them with a current version of Ptolemy II (SVN Development Version). These models are also available in the ptII svn repo as $PTII/src/ptII/org/terraswarm/accessor/demo

  • StockTick: The StockTick.xml Ptolemy II model accesses and displays the stock prices of the publicly-held sponsors of the TerraSwarm project.
  • Hue: The HueToggle.xml Ptolemy II model randomly toggles the color of a Philips Hue lightbulb. The HappySadStock.xml model uses the StockTick accessor to control the color of a Hue lightbulb based on stock prices. You will need to know the IP address of a local Hue Bridge.
  • KeyValueStore: Two Ptolemy II models are provided to demonstrate the KeyValueStore accessor. KeyValueStoreServer.xml provides a server. Run this model to create a key-value storage service on your local host. KeyValueStoreClient.xml provides a simple client that uses this server. More interesting clients will be distributed across several machines.
    You may find our server running and accessible. See http://ptango.eecs.berkeley.edu:8077/keyvalue/help.
