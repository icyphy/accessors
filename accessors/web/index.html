Terraswarm Accessors

Accessors, described in a draft paper, A Vision of Swarmlets, are actors that provide access to a remote service, sensor, or actuator. An accessor is instantiated by an accessor host, which is a program or browser script. The host uses the accessor as if it were a local source and/or sink for data and/or commands. An accessor host is to the Internet of Things what a browser is to the Internet. It renders a remote service by locally executing a proxy for that service.

The wiki page for this workgroup is the place to go to participate in the design of the accessor mechanism.

The TerraSwarm project has developed a "version 0" accessor design and a suite of accessor hosts that should be viewed as concept demonstrations. We are further developing the design. Ptolemy II (the Development Version) is currently the most complete prototype of an accessor host. In Ptolemy II, the accessors listed here can be instantiated using the File-Import-Import Accessor menu item, and can then be incorporated into a Ptolemy II model. For a tutorial on Ptolemy II, see the Ptolemy Book (a free download). To view the accessor script, double click on the accessor instance, select Preferences, and enable "expert mode."

Demos

To run the following models, save them to some directory, and open them with a current version of Ptolemy II (version 10.0 or later).

  • StockTick: The StockTick.xml Ptolemy II model accesses and displays the stock prices of the publicly-held sponsors of the TerraSwarm project.
  • Hue: The HueToggle.xml Ptolemy II model randomly toggles the color of a Philips Hue lightbulb. The HappySadStock.xml model uses the StockTick accessor to control the color of a Hue lightbulb based on stock prices. You will need to know the IP address of a local Hue Bridge.
  • KeyValueStore: Two Ptolemy II models are provided to demonstrate the KeyValueStore accessor. KeyValueStore.xml provides a server. Run this model to create a key-value storage service on your local host. KeyValueStoreClient.xml provides a simple client that uses this server. More interesting clients will be distributed across several machines.
    You may find our server running and accessible. See http://ptango.eecs.berkeley.edu:8077/keyvalue/help.
