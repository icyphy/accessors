Using Subversion to update the accessors web pages

The files that are used to generate the non-wiki and non-JSDoc accessor web pages at http://www.terraswarm.org/accessors/ are checked in to a Subversion (svn) repository.

Only TerraSwarm participants may obtain read/write access to the repository. For details, see Who can get a login?. However, the contents of the repository appear at http://www.terraswarm.org/accessors/ and the repository is available via anonymous access for non-TerraSwarm participants.

This page is updated every five minutes via a cronjob run as the vc user on moog that does a svn update. The output is at svn.out.

The docs are updated by a Jenkins job, see JSDoc Continuous Integration. Note that Jenkins job is not world readable, but the details are world readable.

To update these pages by hand, see How do I update workgroup pages using SVN?

Once you have a repo.eecs.berkeley.edu account and access, you can check out a tree with using read/write access with:

    svn co https://repo.eecs.berkeley.edu/svn/projects/terraswarm/accessors/trunk/accessors

Note that this svn repo is also available via anonymous access:

    svn co https://repo.eecs.berkeley.edu/svn-anon/projects/terraswarm/accessors/trunk/accessors
